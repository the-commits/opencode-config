# AGENTS.md

This document provides guidelines for agentic coding agents operating in this repository.

## Mode-Specific Guidelines

### Analyze Mode

Follow the analysis framework: dissect components, contextualize ecosystem, invert negative space, synthesize patterns, and examine your own lenses. Always use webfetch to read references when analyzing.

### Creative Mode

Suspend judgment, challenge assumptions, cross-pollinate from unrelated fields, use metaphors freely, and embrace constraints as catalysts for innovation.

### Brainstorm Mode

Generate multiple ideas rapidly without evaluation. Focus on quantity and variety of concepts.

### Build Mode

Focus on correctness and efficiency. All tools are enabled for maximum productivity.

### Plan Mode

Think through the entire implementation before acting. Read files and understand the codebase structure before making changes.

### Wild Mode

Maximum creativity with no constraints. Explore unconventional solutions and push boundaries.

## Additional Guidelines

- When editing files, read the file first to understand its structure
- Make minimal, focused changes rather than large refactors
- Follow existing patterns in the codebase unless there's a strong reason to deviate
- Always follow best practices for the project's language and framework
- When in doubt or when no guidance exists, use webfetch to examine references and documentation
- When coding always define tests to verify changed and new code.
- Never use comments other than .i.e paradocs or any equal for the specific framework or language.
- Always follow the best practices regarding naming, but try to always be descriptive.
